## 数据源详情

### 1. 实时数据源（优先）

**数据来源：**
- AKShare - `ak.stock_individual_info_em()`
- AKShare - `ak.stock_zh_a_spot_em()`

**特点：**
- ✅ 实时更新
- ✅ 数据准确
- ✅ 包含实时价格
- ⚠️ 依赖网络连接
- ⚠️ 可能超时或连接失败

**返回字段：**
- `code`: 股票代码
- `name`: 股票名称
- `sector`: 所属行业
- `price`: 最新价格（如果可用）
- `change_pct`: 涨跌幅（如果可用）
- `source`: 数据源标识
- `query_time`: 查询时间
- `data_source`: 'realtime'

---

### 2. 备用数据库（容错）

**数据来源：**
- `modules/stock_codes_db.py` - 经过验证的股票代码数据库

**特点：**
- ✅ 离线可用
- ✅ 数据准确（经过验证）
- ✅ 响应快速
- ⚠️ 不是实时价格
- ⚠️ 需要定期更新

**数据规模：**
- 80+ 经过验证的股票代码
- 涵盖主要行业和龙头股
- 包含ST股票（特别标注）

**返回字段：**
- `code`: 股票代码
- `name`: 股票名称
- `sector`: 所属行业
- `source`: 数据源标识
- `query_time`: 查询时间
- `data_source`: 'backup'
- `note`: '使用备用数据库'

---

### 3. 缓存机制

**缓存策略：**
- 缓存时间：5分钟（300秒）
- 缓存范围：查询结果
- 自动过期：超时自动删除

**特点：**
- ✅ 减少重复查询
- ✅ 提高响应速度
- ✅ 降低API调用
- ⚠️ 数据可能不是最新的

---

## 数据质量保证

### 验证机制

**1. 实时数据验证**
- 网络连接测试
- API响应验证
- 数据完整性检查

**2. 备用数据验证**
- 人工验证所有股票代码
- 定期更新（每季度）
- ST股票特别标注

**3. 数据来源标注**
所有返回数据都包含 `data_source` 字段：
- `realtime`: 实时数据
- `backup`: 备用数据库
- `mock`: Mock数据（仅用于测试）

### 错误处理

**实时数据失败：**
- 自动切换到备用数据库
- 记录警告日志
- 返回数据标注为备用

**备用数据缺失：**
- 返回默认值
- 记录错误日志
- 标注为'未知'

**全部失败：**
- 返回空值
- 记录严重错误
- 触发告警

## 使用示例

### 基本查询

```python
from modules.stock_query_hybrid import StockQueryHybrid

# 初始化查询器（优先实时数据）
query = StockQueryHybrid(prefer_realtime=True)

# 查询单个股票
stock_info = query.query_stock('600519')
if stock_info:
    print(f"股票: {stock_info['name']}")
    print(f"数据源: {stock_info['data_source']}")
```

### 批量查询

```python
# 批量查询
codes = ['000001', '600519', '603019']
results = query.batch_query(codes)

for code, info in results.items():
    if info:
        print(f"{code}: {info['name']} ({info['data_source']})")
```

### 验证股票

```python
# 验证股票代码和名称
result = query.validate_stock('002496')
if result['valid']:
    print(f"✓ {result['code']}: {result['name']}")
    print(f"  数据源: {result['data_source']}")
else:
    print(f"✗ 查询失败: {result['note']}")
```

## 维护和更新

### 备用数据库更新

**更新周期：** 每季度一次

**更新内容：**
- 新上市股票
- 名称变更股票
- 新增ST股票
- 退市股票移除

**更新流程：**
1. 从AKShare获取最新股票列表
2. 人工验证关键股票
3. 更新 `modules/stock_codes_db.py`
4. 运行验证脚本确认

### 验证脚本

## 错误排查

### 实时数据查询失败

**可能原因：**
1. 网络连接问题
2. AKShare API故障
3. 股票代码不存在

**解决方案：**
- 检查网络连接
- 查看错误日志
- 系统会自动切换到备用数据库

### 备用数据库缺失

**可能原因：**
1. 股票代码不在数据库中
2. 数据库未更新

**解决方案：**
- 检查股票代码是否正确
- 更新备用数据库
- 添加缺失的股票代码

---

## 总结

本系统的数据查询策略：

1. **优先实时数据**：确保数据新鲜度和准确性
2. **智能降级**：实时失败自动切换到备用数据库
3. **缓存优化**：减少重复查询，提高性能
4. **透明标注**：所有数据明确标注来源
5. **质量保证**：多重验证机制确保数据准确性

**数据准确性 > 数据实时性**

宁可使用准确的备用数据，也不使用错误的实时数据。
