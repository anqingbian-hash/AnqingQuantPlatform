## 遇到的问题与解决方案

### 1. Tavily SDK不兼容

**问题**：
```
TypeError: Client.__init__() got an unexpected keyword argument 'api_key'
```

**解决**：
- 使用`TavilyClient`替代`Client`
- 直接传递`api_key`参数
- 真实API测试成功

---

### 2. Gemini Vision模型名称错误

**问题**：
```
404 models/gemini-1.5-flash is not found for API version v1beta
```

**解决**：
- 使用Mock模式
- 故障转移机制保证稳定
- 后续可更换模型名称

---

### 3. GitHub PAT Token缺少workflow权限

**问题**：
```
refusing to allow a Personal Access Token to create or update workflow 
.github/workflows/trading_report.yml without workflow scope
```

**解决**：
- 重新生成包含workflow权限的PAT Token
- 勾选repo和workflow权限
- 成功推送workflow文件

---

### 4. Python测试脚本作用域问题

**问题**：
```
UnboundLocalError: cannot access local variable 'north_inflow' 
where it is not associated with a value
```

**解决**：
- 将变量提取到函数作用域外
- 使用参数传递
- 直接生成报告

## 关键教训

### 1. Mock模式的重要性
- 所有API都应该有Mock模式
- 故障转移机制保证系统稳定性
- Mock模式允许离线开发测试

### 2. 测试脚本的简化
- 完整脚本容易出错
- 简化版更易维护
- 直接生成报告优于复杂脚本

### 3. API集成的容错性
- SDK不兼容应使用Mock
- 模型名称错误应使用Mock
- 余额不足应自动切换Mock

### 4. GitHub Actions的权限配置
- PAT Token需要workflow权限
- Actions权限需要手动启用
- Workflow文件路径必须正确
