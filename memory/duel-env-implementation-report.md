## 🏗 系统架构

### 目录结构
```
/unified-quant-platform/
├── app/                          # 应用代码（统一）
│   ├── __init__.py
│   ├── config.py                 # 配置管理器
│   └── routes/
│       └── api.py               # API 路由
├── config/                       # 配置文件
│   ├── config_test.yaml          # 测试环境配置
│   ├── config_prod.yaml          # 正式环境配置
│   └── config.py                # 配置加载器
├── data/                         # 数据目录（环境隔离）
│   ├── test/                    # 测试环境数据
│   │   ├── cache/
│   │   ├── logs/
│   │   └── temp/
│   └── prod/                   # 正式环境数据
│       ├── cache/
│       ├── logs/
│       └── temp/
├── database/                     # 数据库配置
│   ├── db_test.sql              # 测试环境数据库
│   └── db_prod.sql              # 正式环境数据库
├── logs/                         # 运行日志
│   ├── test.log                 # 测试环境日志
│   └── prod.log                # 正式环境日志
└── main.py                       # 主入口（支持环境参数）
```

## 🔒 环境隔离机制

### 完全隔离
1. **数据库**：test/prod 两个独立数据库
2. **缓存**：Redis 使用不同的 DB 索引
3. **日志**：独立的日志文件（test.log/prod.log）
4. **临时文件**：独立的临时目录
5. **配置**：独立的配置文件

### 安全措施
1. ✅ 正式环境关闭调试模式
2. ✅ 正式环境使用生产密码
3. ✅ 正式环境不输出敏感信息
4. ✅ 定期备份正式环境数据

## 🚀 使用方法

### 启动测试环境
```bash
# 方法1：使用环境变量
export APP_ENV=test
python main.py

# 方法2：使用命令行参数
python main.py --env=test

# 服务地址
http://localhost:5000
```

### 启动正式环境
```bash
# 方法1：使用环境变量
export APP_ENV=prod
python main.py --env=prod

# 方法2：使用命令行参数
python main.py --env=prod

# 服务地址
http://localhost:8000
```
