# 2026-03-05 R2检测器优化进度

## R2检测器版本演进

### V5-V8：网格线问题
- **问题**：所有版本都选择y=84-95的超长网格线（length=2555px）
- **准确率**：0%-40%（2/5）
- **平均误差**：7.8%-8.2%

### V9：网格线检测与避免（当前版本）
- **改进**：成功避免网格线，所有候选都没有网格线
- **准确率**：60%（3/5）
- **平均误差**：5.2%
- **问题**：选择了过短的线条（100-200px），真正的R2线是中等长度（300-900px）

### 关键发现

1. **R2线颜色**：黑色（brightness=3.4-17.6）
   - 所有5个周期的R2线都是黑色
   - 之前的假设"白色R2线"是错误的

2. **R2线长度**：300-900px（中等长度）
   - 周线：y=207，附近最大长度2555px（有网格线）
   - 日线：y=306，附近最大长度733px（无网格线）
   - 1小时：y=445，附近最大长度653px（无网格线）
   - 15分钟：y=562，附近最大长度2555px（有网格线）
   - 5分钟：y=225，附近最大长度2555px（有网格线）

3. **网格线特征**：2555px超长线，所有周期都有
   - 网格线位置：y=84-91（上部）
   - 网格线颜色：灰色/黑色

4. **V9评分权重**：
   - Grid Penalty Weight: 0.4（过高）
   - Max Line Length Weight: 0.25
   - Brightness Weight: 0.15
   - Position Weight: 0.1
   - Color Weight: 0.1

### 下一步：V10优化方向

1. **降低网格线惩罚权重**：0.4 -> 0.2
   - 原因：R2线可能与网格线共存，不能完全惩罚

2. **提高线条长度权重**：0.25 -> 0.35
   - 原因：中等长度（300-900px）是R2线的关键特征

3. **添加边缘密度权重**：新增
   - 原因：R2线应该有明显的边缘（edge_density较高）

4. **目标线条长度**：300-900px
   - 周期调整不同周期的最优长度范围

### 待解决问题

1. **周期特异性**：不同周期的R2线长度分布不同
   - 日线、1小时：无网格线，最大长度500-700px
   - 周线、15分钟、5分钟：有网格线，需要更智能的选择

2. **多候选融合**：当前只选择最高分的一个候选
   - 可以考虑多个候选的加权平均

3. **Y坐标映射精度**：当前R²=0.25-0.69
   - 可以优化线性回归模型或使用非线性模型
