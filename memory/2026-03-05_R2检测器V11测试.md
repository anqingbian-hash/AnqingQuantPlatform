- **根因**：y=207附近的R2线是短线条（196px），被长度评分惩罚（length_score=0.5）
- **候选分析**：
  - y=216: edge_density=0.201, length=2555, length_score=0.5, 综合评分=0.200
  - y=129: edge_density=0.187, length=463, length_score=0.8, 综合评分=0.287

### 下一步优化方向（V12）

1. **调整长度评分策略**
   - 当前：200-1500px范围
   - 改进：降低短线条惩罚（200-300px也给予较好评分）
   - 原因：正确R2线可能是短线条（196px）

2. **降低edge_density阈值**
   - 当前：>=0.10给予0.9分
   - 改进：>=0.08给予0.9分
   - 原因：正确R2线的edge_density可能较低（0.095）

3. **增加周期特异性**
   - 当前：所有周期使用同一参数
   - 改进：不同周期使用不同长度范围
   - 原因：周线的R2线较短（196px），其他周期较长（500-700px）

4. **考虑Y坐标映射误差**
   - 当前：Y坐标映射R²=0.25-0.69
   - 改进：优化映射模型，减少Y->Price的转换误差

1. **Edge Density是关键特征**
   - 正确R2位置的edge_density：0.075-0.118
   - 伪线y=128的edge_density：0.091-0.093
   - V11通过提高edge_density权重（0.25）成功避开伪线

   - 但R2线仍有较高的edge_density（0.075）
   - 结论：不能完全惩罚有网格线的位置
