### V12失败原因

#### 核心问题：所有周期都检测到y=120
- y=120在所有周期都有相同的特征：
  - brightness=15-16（黑色）
  - edge_density=0.187（非常高）
  - length=463px（中等长度）
- 这不是R2线，而是**SR线（动态滤网）**

#### y=120是什么线？
根据分析：
1. **SR线（动态滤网）**：支撑/压力线
2. 根据memory：SR线的Y坐标都是122（所有周期）
3. y=120与SR线位置（122）非常接近
4. 结论：y=120是SR线，不是R2线

#### 为什么V12选择了y=120？
1. **edge_density过高**：0.25（V12: 0.25，所有版本中最高）
2. **y=120的edge_density=0.187**：非常高，远超其他候选
3. **位置权重不足**：0.10（不足以约束R2线应该在SR线之上）

#### R2线应该在什么位置？
- SR线位置：y=122（所有周期）
- R2线应该在SR线之上：y < 122
- 但正确R2线实际位置：
  - 周线：207（在SR线之下！）
  - 日线：306（在SR线之下！）
  - 1小时：445（在SR线之下！）
  - 15分钟：562（在SR线之下！）
  - 5分钟：225（在SR线之下！）

**结论**：R2线实际上在SR线之下，不是之上！

### 重大发现：R2线与SR线的关系

根据实际数据：
- **SR线（动态滤网）**：y=122（所有周期）
- **R2线（阻力线）**：
  - 周线：y=207（SR线之下85px）
  - 日线：y=306（SR线之下184px）
  - 1小时：y=445（SR线之下323px）
  - 15分钟：y=562（SR线之下440px）
  - 5分钟：y=225（SR线之下103px）

**这意味着**：
1. R2线在SR线之下
2. R2线是"阻力位"，SR线是"支撑/压力位"
3. 可能是我的理解错了：R2线应该低于SR线

### 重新理解R2线

根据memory中的数据：
```
detected_y_coords: {
  "r2_y": 207,  // 周线
  "s2_y": 122,  // SR线（支撑线）
  "current_y": 122  // 现价线
}
```

这说明：
- R2线（207）> S2线（122）
- 在图表坐标系中，y越大越靠下
- 所以R2线在S2线之下

**我的错误假设**：
- ❌ 假设：R2线应该在SR线之上（y < 122）
- ✅ 实际：R2线在SR线之下（y > 122）
