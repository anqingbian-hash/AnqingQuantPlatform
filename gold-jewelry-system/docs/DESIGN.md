## 一、系统概述

### 1.1 项目背景

中国贵金属首饰市场规模高达 5000 亿元/年，但中小珠宝零售企业的数字化管理水平普遍较低，存在库存管理混乱、价格滞后、防盗防损困难等痛点。

本系统专为中小型珠宝零售企业打造，提供智能化、SaaS 化的管理解决方案。

### 1.2 核心价值

1. **防损增效**：库存精准率从 70% 提升到 98%，减少损失 5-10%
2. **智能定价**：实时金价联动，自动计算售价，提升毛利 2-3%
3. **客户体验**：一物一码溯源，真伪可查，提升客户信任度

### 1.3 目标客户

- 主要：1-20 家分店的珠宝连锁品牌
- 次要：年销售额 500 万以上的独立珠宝店

## 四、核心业务逻辑

### 4.1 入库流程

```
1. 用户录入商品信息
   ↓
2. 系统校验商品编号唯一性
   ↓
3. 获取实时金价
   ↓
4. 计算入库成本
   成本 = (金价 × 金重 × 成色 / 100) + 工费
   ↓
5. 更新库存表
   ↓
6. 记录库存流水
   ↓
7. 发送消息（同步多店）
```

### 4.2 销售出库流程

```
1. 店员扫码
   ↓
2. 系统显示商品信息和实时售价
   ↓
3. 确认收银
   ↓
4. 扣减库存（乐观锁）
   ↓
5. 生成销售订单
   ↓
6. 记录库存流水
   ↓
7. 发送消息（同步多店）
```

### 4.3 库存调拨流程

```
1. A 店创建调拨申请
   ↓
2. 冻结 A 店库存
   ↓
3. B 店确认调拨
   ↓
4. A 店出库
   ↓
5. 物流运输
   ↓
6. B 店入库
   ↓
7. 数据同步
```

## 五、安全机制

### 5.1 认证机制

**JWT（JSON Web Token）认证流程：**

1. 用户登录，验证用户名和密码
2. 生成 JWT Token（有效期 24 小时）
3. 客户端存储 Token（localStorage）
4. 每次请求携带 Token（Header: Authorization: Bearer {token}）
5. 服务端验证 Token，提取用户信息

### 5.2 权限控制

**RBAC（Role-Based Access Control）权限模型：**

| 角色 | 权限 |
|------|------|
| 超级管理员（role_id = 1） | 全部权限 |
| 店长（role_id = 2） | 商品管理、库存管理、销售管理、调拨管理、报表查看 |
| 店员（role_id = 3） | 扫码销售、库存查询 |
| 财务（role_id = 4） | 财务报表、数据导出 |

### 5.3 数据加密

1. **密码加密：** BCrypt（不可逆加密）
2. **敏感字段加密：** AES-256
3. **传输加密：** HTTPS + TLS 1.3

### 5.4 操作日志

所有操作记录日志，包括：
- 操作人 ID
- 操作时间
- 操作模块
- 操作类型
- 操作内容
- IP 地址
- User-Agent

## 六、性能优化

### 6.1 缓存策略

| 数据 | 缓存时间 | 说明 |
|------|----------|------|
| 商品列表 | 5 分钟 | 减少数据库查询 |
| 库存数据 | 10 秒 | 实时性要求高 |
| 金价数据 | 1 小时 | 定时更新 |
| 用户信息 | 30 分钟 | 减少 Redis 访问 |

### 6.2 数据库优化

1. **索引优化：** 所有查询字段添加索引
2. **查询优化：** 避免 SELECT *，只查询需要的字段
3. **分页查询：** 使用 Spring Data JPA 分页
4. **慢查询监控：** 开启 SQL 日志，定期分析

### 6.3 代码优化

1. **异步处理：** 使用 @Async 异步执行耗时操作
2. **批量操作：** 使用 JPA 批量插入/更新
3. **连接池：** HikariCP（默认）
4. **对象池：** 复用对象，减少 GC 压力
